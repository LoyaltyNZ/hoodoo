    class ApiTools::ServiceRequest

    Instances of the ApiTools::ServiceRequest class are passed to service interface implementations when requests come in via Rack, after basic checks have been passed and a particular interface implementation has been identified by endpoint.

    Descriptions of default values expected out of accessors herein refer to the use case when driven through ApiTools::ServiceMiddleware. If the class is instantiated “bare” it gains no default values at all (all read accessors would report nil).

    Attributes

    body[RW]

    Parsed payload hash, for create and update actions only; else nil.

    embeds[RW]

    Array of strings giving requested embedded items; [] if there are none requested.

    ident[R]

    The first entry in the uri_path_components array, or nil if the array is empty. This supports a common case for inter-service calls where a UUID or other unique identifier is provided through the first path element (“.../v1/resource/uuid”).

    list_filter_data[RW]

    List filter key/value pairs as a hash, all keys/values strings; {} if there's no filter data in the request URI query string.

    list_limit[RW]

    List page size, for index views; an integer; always defined.

    list_offset[RW]

    List offset, for index views; an integer; always defined.

    list_search_data[RW]

    List search key/value pairs as a hash, all keys/values strings; {} if there's no search data in the request URI query string.

    list_sort_direction[RW]

    List sort direction, for index views; a string; always defined.

    list_sort_key[RW]

    List sort key, for index views; a string; always defined.

    locale[RW]

    Requested locale for internationalised operations; +“en-nz”+ by default.

    references[RW]

    Array of strings giving requested referenced items; [] if there are none requested.

    uri_path_components[R]

    An array of zero or more path components making up the URI after the service endpoint has been accounted for. For example, with a service endpoint of “products”, this URI:

    http://test.com/products/1234/foo.json

    …would lead to this path component array:

    [ '1234', 'foo' ]

    The first element of the path components array is exposed in the read-only ident accessor.

    uri_path_extension[RW]

    A filename extension on the URI path component, if any, else an empty string. The first dot in the last path component is looked for (see also uri_path_components), so for example this URI:

    http://test.com/products/1.2.3.4/foo.my.tar.gz

    …would lead to this URI path extension string:

    'my.tar.gz'

    ary

    Path component array to record. If nil or not an array, nil is stored for #uri_path_components and ident.

    # File lib/api_tools/service_implementations/service_request.rb, line 58
def uri_path_components=( ary )
  if ary.is_a?( ::Array )
    @uri_path_components = ary
    @ident               = ary.first
  else
    @uri_path_components = nil
    @ident               = nil
  end
end
